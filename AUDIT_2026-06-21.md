# Pokémon Dealer OS — Audit & Recommendations (2026-06-21)

## What was broken

Your repro case — name `Charizard`, number `04/102`, set `base set` — returned nothing. Root causes, all in `src/lib/catalog/pokemonTcgApi.ts`:

1. **Collector numbers weren't normalized.** The live Pokémon TCG API stores plain numbers without leading zeros (`4`, not `04`). You typed `04/102`; the query sent `number:"04/102"` verbatim and never matched.
2. **Set name was a brittle phrase match.** `"base set"` was sent as `set.name:"base set"` — a literal phrase query against the API's actual set name, `"Base"`. No fuzzy or alias handling, so any nickname, abbreviation, or partial name failed.
3. **No fallback.** One query, all-or-nothing. If any single field (name, number, or set) didn't match exactly, you got zero results — even though name+number alone was enough to find the card.
4. **Scoring bug.** The card-ranking logic gave a bonus for set names that merely *contained* the query as a substring, which could rank a wrong, longer-named set above the correct one.

## What was fixed

- **New offline set catalog** (`src/lib/catalog/setCatalog.ts`): all 173 official sets bundled locally, each resolvable by exact ID, exact name, PTCGO code, or curated nicknames (`"base set"`, `"1st edition base set"`, `"151"` → `sv3pt5`, `"hgss"`, etc.) via `resolveSetId()`. Works with zero API calls or keys.
- **Progressive query relaxation**: `resolve()` now tries name+number+set, then name+set, then name+number, then name alone — stopping at the first level that returns results. Your exact case now resolves correctly on the second attempt.
- **Leading-zero normalization**: `04/102` → `4` before querying; alphanumeric codes (`TG05`, `SWSH001`) are left untouched since those *do* keep their padding on the live API.
- **Set resolution uses `set.id` exact match** instead of fragile `set.name` phrases, via the new catalog.
- **Scoring fixed**: dropped the substring-bonus bug; matches now compare exactly against the resolved set ID.
- **UX, per your request to pre-load from the API**: added `/api/catalog/sets` (popular-sets list + full 173-set list) and `/api/catalog/search` (fuzzy set search), both backed by the bundled catalog. The Set field in the Acquire form now has a live autocomplete dropdown as you type, plus a row of one-click "popular set" chips (Base, 151, latest sets, etc.) so you rarely have to type a set name correctly at all.

All of this is offline — no new API key requirements, no live calls needed to search/resolve sets.
