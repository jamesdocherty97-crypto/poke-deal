# Poke Deal — Visual Bug Audit

**Date:** 2026-07-04
**Method:** Live click-through of https://poke-deal.vercel.app in Chrome. Observation only — nothing was published, deleted, or sold; every destructive confirm dialog encountered was cancelled. One card was added to stock (Pikachu, Crown Zenith Galarian Gallery, PSA 9, £15–£25 cost) to reach "add-to-session" and sheet states that don't exist without stock; no listing was published and no sale was completed.
**Viewports tested:** ~606×713 (devicePixelRatio 2) for the mobile pass — the environment's `resize_window` tool would not hold the requested 390×844 this session, so this narrower-but-imperfect viewport was used instead and is noted wherever it may be relevant. Desktop pass at 1512×827 (browser default, unresized).
**Card types rotated:** graded slab (Pikachu PSA 9) and raw card (Snivy / Pansage / Infernape, RAW).

Numbering is continuous across all findings. Table is sorted by severity: blocker → annoyance → nitpick.

## Findings

| # | Severity | Screen / State | Type | Description |
|---|----------|-----------------|------|--------------|
| 1 | Blocker | Inventory row, "More" action strip (any row) | OVERFLOW | Expanding "More" on a stock row reveals an 11-button strip (Photos, Image URL, Comp, Copy eBay, Copy CM, Copy Vinted, Edit, Sell, Draft, Hold, Delete) that overflows the viewport with no visible scrollbar or fade/arrow hint. Normal mouse-wheel horizontal scroll over the strip does not move it at all; the later buttons (Draft, Hold, Delete) were only reachable by asking the browser to scroll a specific element into view. A user relying on touch/wheel scroll has no way to discover or reach Delete, Hold, or Draft. |
| 2 | Blocker | Inventory list, any row, at desktop width (1512px) | OVERLAP | The fixed bottom nav bar (Buy/Inventory/Listings/P&L/Setup) sits directly on top of the action-button area of stock rows in the two-column desktop layout, completely hiding the "Draft listing"/action buttons on both visible cards at once. Only the swipe-label ghosts ("SELL" / "DELETE", see #7) are visible peeking out past the nav's left/right edges. |
| 3 | Annoyance | Buy → "Ready to stock / Add cost" panel, after typing a cost | OVERLAP | The sticky bottom result bar (e.g. "Single graded £37.44 / Buy / Watch / Pass") sits directly on top of the "Profit" field's value, so the computed profit is completely invisible until the user manually scrolls past the sticky bar — the field label ("Profit") is visible but the number itself is not. |
| 4 | Annoyance | Buy screen, any resolved-comp state (Manual card / Manual check / Single graded) | OVERLAP | The sticky bottom action bar consistently overlaps the content immediately above and below it: it clips the rounded border of the "HEARD" summary panel above, and sits flush against (in some states, appears to touch/clip) the grade-selector button grid and the bottom nav. Reproduced across three separate resolved states in this session. |
| 5 | Annoyance | Inventory → Sell sheet ("Mark sold") and Edit sheet ("Edit stock"), any row | OVERLAP / ILLEGIBLE | Both sheets use a backdrop that is not opaque enough — the text and values of the row(s) underneath (card name, set, cost, "6d old", etc.) remain legible, ghosted behind the sheet's own fields (e.g. behind the OFFERS button row in Sell, behind the Cost/Qty fields in Edit). Reproduced in two different sheets and two different sale outcomes (loss and profit). |
| 6 | Annoyance | Inventory → Sell sheet ("Mark sold") | INCONSISTENT | Multiple "Create sale" call-to-action buttons are shown at once for the same action: one inline inside the green/red "SALE STEP" verdict card, and a second (plus, in the profit-path variant, a third "Save + next") pinned at the bottom of the sheet. It's unclear which is the "real" primary action. |
| 7 | Annoyance | Inventory row, resting state (no swipe gesture) | OVERLAP | A "SELL" label (left) and a "DELETE" label (right) — presumably swipe-to-reveal action backgrounds — are visibly bleeding through around the outside of the row's rounded card border at all times, without any swipe having been performed. |
| 8 | Annoyance | Buy → success toast after "Stock with caution" | OVERLAP | The green "Stocked. Listing skipped." confirmation toast renders on top of the sticky action bar it replaces, overlapping the card's price text (partially obscuring "£37.44", leaving only "£3" visible) and the "Pass" button. |
| 9 | Annoyance | Listings → open a draft's "Listing pack" | OVERLAP | Same backdrop bleed-through bug as #5: "Listing desk" / "Sales desk" text from the page underneath is legible behind the "Net after sale" panel of the Listing pack sheet. |
| 10 | Annoyance | Listings → "Listing pack" → next-step guide (Review pack / Create offer / Publish live / Book sale) | CLUNKY | The 4-step guide is styled like a clickable wizard (numbered circular badges, coloured borders, step 2 highlighted) but none of the steps respond to clicks — they are purely informational text. Because eBay API automation is currently disabled for this account, the app substitutes a manual "Copy + open" flow, but nothing on screen explains that the numbered steps are non-interactive; a user could reasonably expect to tap through them to reach the "Publish live" confirmation. In this state the in-app publish confirmation described in the brief is not reachable at all. |
| 11 | Annoyance | Setup → Status page (accessed via the top-right "Status" button) | INCONSISTENT | Opening Status changes the page header to "Status" and its own content, but the bottom nav bar keeps "Setup" highlighted as the active tab, which no longer matches what's on screen. |
| 12 | Annoyance | Bottom nav bar, desktop width (1512px), any screen | CLUNKY | The nav does not adapt to the wider viewport — it stays a narrow, mobile-proportioned floating pill roughly centred at the bottom of the screen, with large unused margins on both sides, rather than spanning the width or relocating (e.g. to a sidebar) the way the rest of the desktop layout seems intended to. |
| 13 | Annoyance | Buy → Smart comp search, typing a query with grade info (desktop width, e.g. "Pikachu, Crown Zenith Galarian Gallery, GG30/GG70, PSA 9") | CLUNKY | While typing, an autosuggest match card ("Pikachu Crown Zen… / Fill / Comp") pops up mid-keystroke. In this run the resulting parsed CARD field read "Pikachu Crown Zenith SA 9" (missing the leading "P" of "PSA") and GRADE fell back to RAW instead of auto-selecting PSA 9 — the same query correctly detected PSA 9 on the mobile pass. This could send the user into the wrong pricing tier (raw vs. graded). Flagged with the caveat that a dropped keystroke could be an artifact of scripted/fast typing rather than genuine human typing speed, but the underlying "typing while a popup mounts" interaction is worth a human re-check. |
| 14 | Annoyance | Inventory row → "More" → "Photos" | CLUNKY | Clicking "Photos" produces no visible in-page change of any kind (no sheet, no toggle state, no loading indicator). If it's meant to open a file picker, there's no on-screen confirmation that anything happened. |
| 15 | Nitpick | Buy screen, empty state, Smart comp search placeholder | OVERFLOW | The placeholder example text ("Umbreon prismatic, Victini promo, Lugia Neo Genesis CGC 1.5") is wider than the input and gets hard-clipped at the field's right edge rather than truncating with an ellipsis or wrapping. |
| 16 | Nitpick | Buy screen, "Recent Sets" / "Popular Sets" chip rows | OVERFLOW | Horizontally-scrolling chip rows end abruptly at the edge of the viewport with no fade, arrow, or other affordance to indicate more content is scrollable. |
| 17 | Nitpick | Buy screen, Smart comp search field | CLUNKY | Ctrl+A followed by typing did not reliably replace the full field contents — on two occasions leftover characters from a previous query remained mixed into the new one. Triple-click reliably selects the full value and was used as a workaround for the rest of this audit. Possibly an automation-environment quirk rather than a genuine browser bug; worth a quick manual check. |
| 18 | Nitpick | Buy screen, after a completed buy (e.g. post-stocking), new empty query | INCONSISTENT | The "Current card / Next card" preview shows a leftover queue entry with a grade badge (e.g. "PSA 9") but no card name and an empty image placeholder — a half-populated ghost entry rather than either a filled entry or nothing. |
| 19 | Nitpick | Buy screen, typing a raw card query (e.g. "Snivy, Scarlet and Violet 151, 6/165") | MISALIGNED | The live "HEARD" parse absorbed the set-name tokens into the CARD field ("Snivy Scarlet and Violet") rather than separating them cleanly into CARD vs SET, even though SET and NO. were separately and correctly detected below it. |
| 20 | Nitpick | Setup → Deal calculator section | MISALIGNED | The "Include promoted listing fee" checkbox renders oversized and centred on its own line above the label text, rather than inline beside ("Include promoted listing fee ☑") like a normal labelled checkbox. Reproduced after a full page reload, so it's a persistent layout issue, not a one-off render glitch. |

## Not reproduced this session
- A distinct "comp loading" spinner state — comps in this environment resolved near-instantly or returned "No auto comp" / "Manual check" rather than visibly loading.
- A live in-app "Publish live" confirmation overlay in Listings — the eBay connection for this account currently has automation disabled (see finding #10), so the pack sheet only offers a manual copy/paste path, not a one-tap publish-to-confirmation flow.
- P&L (all tables) and the main Setup form fields (Listing copy, Postage tiers, Confidence haircuts, Grading EV) were walked in full and showed no layout defects beyond #20.
