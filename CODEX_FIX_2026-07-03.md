## F2 — PokeTrace market-fallback cooldown poisoning (root cause of the sweep's zero corroboration)
Diagnosis (design-layer confirmed): adapter tries `market=EU` before `market=US`; the Free plan is US-only, so EU returns 403; `fetchWithRetry` classifies ANY 403 as key-problem and enters the shared 10-minute cooldown BEFORE the US fallback executes. One plan-restricted market poisons the entire source. Evidence: Snivy succeeded on a fresh instance; dashboard shows 11/250 calls; every sweep probe short-circuited.
1. Scope 403 handling per-market: a 403 on market M means "M not permitted" — record it in a per-market cooldown/deny memo (long TTL, e.g. 6h) and FALL THROUGH to the next market immediately. Only when ALL configured markets 403 does the source enter the key-problem cooldown and surface in Setup health.
2. Make market order configurable: `POKETRACE_MARKETS` env (comma list), default `"US,EU"` (US first — matches both the current Free plan and the reconciler's treatment of PokeTrace as a US baseline). Also SET `POKETRACE_MARKETS=US` in Vercel production and document in .env.example that Pro-plan accounts can use `EU,US`.
3. Verify the candidate's region/market labeling: data fetched with market=US must be labeled US, market=EU labeled EU, so the reconciler's region factor stays truthful whichever market answered.
4. 429 handling unchanged (Retry-After + short cooldown is correct for rate limits).
5. Tests: EU-403→US-success yields a priced US-labeled candidate with NO global cooldown; all-markets-403 → key-problem cooldown + Setup health; per-market deny memo skips EU on subsequent calls; env order respected.
6. Deploy, then live-verify: Gengar LOR TG06 RAW via prod /api/comps must show a PokeTrace row with a price (was "key problem" all sweep).
